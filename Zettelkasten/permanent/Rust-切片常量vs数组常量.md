## 1. 核心观点  
> `&[usize]` 常量更高效（避免拷贝）、更灵活（可直接传给 slice 参数）。

## 3. 展开说明  
### 核心区别分析

| 项目       | `[usize; 9]` 数组常量        | `&[usize]` 切片常量                     |
| -------- | ------------------------ | ----------------------------------- |
| **类型**   | 固定长度数组                   | 不定长切片引用                             |
| **内存分配** | **内联到每个使用点（复制）**         | 指向静态内存中唯一的常量数组                      |
| **大小**   | `9 * size_of::<usize>()` | 指针+长度（16字节在64位机上）                   |
| **传递开销** | 可能复制整数组                  | **仅传递一个引用（轻量）**                     |
| **灵活性**  | 类型固定 `[usize; 9]`        | **通用 `&[usize]`，可用于动态长度接口**         |
| **适用接口** | 无法直接传入 `&[usize]` 参数     | 可直接传入切片函数，如 `fn f(slice: &[usize])` |
