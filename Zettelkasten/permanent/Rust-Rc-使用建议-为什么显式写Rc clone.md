# 为什么要显式写 `Rc::clone(&a)` 而不是 `a.clone()`
## 1. 核心观点  

###  （1）语义清晰
- `a.clone()`：容易让人误以为是**深拷贝整个数据**；
- `Rc::clone(&a)`：明确表明是**Rc 层面的计数+1**，不涉及数据复制。
- 原因见展开说明

> Rust 倡导**语义透明的克隆调用**。  
> 对数据本身的深度克隆用 `.clone()`；  
> 对引用计数的共享克隆用 `Rc::clone(&x)`。

## 3. 展开说明  
###   语义清晰的原因。一眼区分“复制数据” vs “共享数据”

|调用形式|语义|性能|数据副本|
|---|---|---|---|
|`a.clone()`|深拷贝整个数据|高成本|多份数据|
|`Rc::clone(&a)`|增加引用计数|极低成本|一份数据（共享）|
