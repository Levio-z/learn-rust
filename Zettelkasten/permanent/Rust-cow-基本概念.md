## 1. 核心观点  
`Cow` 的核心思想是“写时克隆”**。**只有在数据需要被修改时，它才会进行克隆操作，而在数据没有被修改时，它会保持为借用（只读）**。
- 当你只需要**读取**数据时，`Cow` 只会借用数据，避免不必要的内存分配和克隆。
- 当你需要**修改**数据时，如果 `Cow` 借用着数据，它会先自动克隆一份，然后对克隆出的新数据进行修改。这样既保证了修改的安全性（不影响原始数据），又实现了性能优化（只有在需要修改时才进行克隆）。如果它已经拥有了数据（即数据是它自己的），则直接在原地修改，不会产生额外的克隆。

## 3. 展开说明  
```
pub enum Cow<B> 
where B: ToOwned + ?Sized { 
Borrowed(&'a B), Owned(B::Owned), 
}
```
- **Borrowed**：表示数据是以借用的方式存在的，数据本身没有被克隆，只有对其的引用。
- **Owned**：表示数据已经被克隆并拥有自己的所有权。
