## 1. 核心观点  
>**定义：** 这是一个用于**标识（而不是定位或交互）任何抽象或物理资源的字符串**。

**统一资源标识符**，英语：**U**niform **R**esource **I**dentifier，缩写：**URI**）是在互联网中唯一的字符序列用于**标识各类抽象或物理资源等对象的统一符号和编码规则**，例如网页上的资源、邮件地址、电话号码、书籍和现实世界的对象如人和地点、概念等。

该种标志**允许用户对网络中（一般指[万维网](https://zh.wikipedia.org/wiki/%E4%B8%87%E7%BB%B4%E7%BD%91 "万维网")）的资源通过特定的[协议](https://zh.wikipedia.org/wiki/%E7%B6%B2%E7%B5%A1%E5%82%B3%E8%BC%B8%E5%8D%94%E8%AD%B0 "网络传输协议")进行交互操作**。URI的最常见的形式是统一资源定位符（URL），经常指定为非正式的网址。更罕见的用法是[统一资源名称](https://zh.wikipedia.org/wiki/%E7%BB%9F%E4%B8%80%E8%B5%84%E6%BA%90%E5%90%8D%E7%A7%B0 "统一资源名称")（URN），其目的是通过提供一种途径。用于在特定的[命名空间](https://zh.wikipedia.org/wiki/%E5%91%BD%E5%90%8D%E7%A9%BA%E9%97%B4 "命名空间")资源的标志，以补充网址。

Uniform规定统一的格式可方便处理多种不同类型的资源，而不用根据上下文环境来识别资源指定的访问方式。另外，加入新增的协议方案（如http：或ftp:）也更容易。

Resource资源的定义是“可标识的任何东西”​。不仅是文档文件，图像或服务（例如当天的天气预报）等能够区别于其他类型的，全都可作为资源。另外，资源不仅可以是单一的，也可以是多数的集合体。

Identifier表示可标识的对象。也称为标识符。

综上所述，URI就是由**某个协议方案表示的资源的定位标识符**。协议方案是指访问资源所使用的协议类型名称。

| 特征       | URN (Uniform Resource **Name**) | URL (Uniform Resource **Locator**)  | URI (Uniform Resource **Identifier**) |
| -------- | ------------------------------- | ----------------------------------- | ------------------------------------- |
| **关注点**  | **资源是什么**（资源的身份/名称）             | **资源在哪里**（资源的位置/如何获取）               | **如何标识资源**（涵盖名称和位置）                   |
| **目的**   | **持久性、唯一标识**，即使资源位置改变。          | **寻址、访问**资源。                        | **统一标识**任何资源。                         |
| **核心特点** | 基于命名空间，通过名称指出资源。                | 基于协议和地址，指出资源位置。                     | **URN和URL都是URI。**                     |
| **举例**   | `urn:isbn:0451450523` (ISBN书号)  | `https://www.example.com/page.html` | 涵盖以上两者                                |

## 3. 展开说明  
URI 作为一个总称，包含了两种子集：

| 类型       | 统一资源定位符（URL）                               | 统一资源名称（URN）                                 |
| -------- | ------------------------------------------ | ------------------------------------------- |
| **英文缩写** | **U**niform **R**esource **L**ocator       | **U**niform **R**esource **N**ame           |
| **核心目的** | **定位 (Locating)** 资源                       | **命名 (Naming)** 资源                          |
| **功能**   | 告诉用户**如何找到**资源以及**如何与之交互**（包含访问机制和网络位置）。   | 告诉用户资源的**身份**或**名称**，而不考虑其当前位置。             |
| **你提到的** | **最常见的形式**，**非正式的网址**。                     | **更罕见的用法**，**通过提供一种途径**（在特定的命名空间内标识资源）。     |
| **示例**   | `https://www.google.com/` (`http` 协议 + 位置) | `urn:isbn:978-7-121-12503-4` (标识一本书，无论它在哪里) |
| **关系**   | URL **包含**定位信息，因此可以用于**交互操作**。             | URN **补充**了 URL，专注于资源的**持久身份**。             |
**统一资源标识符**，英语：**U**niform **R**esource **I**dentifier，缩写：**URI**）是在互联网中唯一的字符序列用于标识各类抽象或物理资源等对象的统一符号和编码规则，例如网页上的资源、邮件地址、电话号码、书籍和现实世界的对象如人和地点、概念等。

**URI = Scheme (协议名) + `:` + Scheme-Specific Part (协议对应的内容)**
- **Scheme (协议名):** 定义了用于访问资源的方法或命名空间（例如 `http`, `https`, `ftp`, `mailto`, `file`, `urn` 等）。
- **冒号 (`:`):** 作为分隔符，将协议名与协议对应的内容分开。
- **Scheme-Specific Part (协议对应的内容):** 这是URI的核心部分，其具体结构取决于前面的协议名

**通用URI的格式如下：**
```
[协议名]://[用户名]:[密码]@[主机名]:[端口]/[路径]?[查询参数]#[片段ID]
```

特定的协议定义了协议内容的语法和[语义](https://zh.wikipedia.org/wiki/%E8%AF%AD%E4%B9%89 "语义")，而所有的协议都必须遵循一定的URI文法通用规则，亦即为某些专门目的保留部分特殊字符。
它强调了两个关键点：

1. **协议的特异性 (Syntax and Semantics)：**
    
    - 每一个具体的协议（如 HTTP、FTP、mailto 等）都有其独特的**语法**（如何构造其内容）和**语义**（这些内容代表什么意义）。
        
2. **URI文法的通用性 (General Rules)：**
    
    - 尽管每个协议有其特定性，但所有协议的 URI 都必须遵循一套**URI 文法通用规则**。
        
    - **特殊字符的保留（Reserved Characters）：** 这套通用规则定义了哪些字符是为了**专门目的**而保留的（例如，`:`, `/`, `?`, `#`, `@`, `&`, `=` 等）。这些字符在 URI 中具有结构性的意义，如果它们需要作为数据的一部分出现，就必须使用百分号编码。
        

**总结：** 这一机制确保了 URI 既能满足特定协议的需求，又能在整个互联网上保持统一的结构和可解析性。
