## 1. 核心观点  

IO 是输入/输出的首字母缩写词。它意味着**从程序到程序之外的世界的任何通信**。
- 具体可能是读取或写入磁盘或网络、写入终端、从键盘或鼠标获取用户输入，或者**与操作系统或系统中运行的其他程序进行通信**。
**IO 在并发性方面很有趣**
- 因为它比程序在内部可能执行的几乎任何任务都要长几个数量级。这通常意味着大量的等待，而等待时间是做其他工作的机会。

## 3. 展开说明  
### IO操作何时完成
从**程序视角**看，IO 在**系统调用返回**时被认为“完成”；从**系统与硬件视角**看，这通常只表示**责任已转移**，而非数据已到达最终介质。

- **系统调用返回**：内核已接管请求（拷贝到内核缓冲、提交队列、或交给驱动）。
    
- **内核完成**：驱动已把请求提交给设备或 DMA 队列。
    
- **设备完成**：数据进入设备缓存或控制器缓存。
    
- **介质可见**：数据真正落盘、显示、被对端程序读取。  
    结论：**“完成”是多层语义，不同 API 承诺不同层级。**


#### 写路径的真实流程（以磁盘/显示为例）

- **用户态 → 内核态**：`write()` 返回，通常表示数据已进入**内核页缓存**或被排队。
    
- **缓存与合并**：内核可能延迟、合并写入（write-back）。
    
- **驱动/控制器**：请求进入设备队列，可能只到**设备缓存**。
    
- **介质提交**：磁盘写磁、GPU 扫描出帧，才是最终可见。  
    关键点：**返回 ≠ 落盘/可见**。

### 线程与io
- 嵌入式：线程可以通过循环_等待_直到 I/O 操作完成
