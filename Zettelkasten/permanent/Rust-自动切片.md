## 核心观点  
> Rust 中的 **自动切片**（auto-slicing）是语言在方法调用和集合类型（如 `Vec<T>`、`String`、数组 `[T; N]`）使用上的一个便利特性，解决在在**方法调用时的借用便利性问题**，它经历了从早期**内置硬编码**到现代 Deref + 自动引用/解引用机制的演进。

## 展开说明  

### **定义**：  
自动切片指的是在方法调用时，编译器会**自动将集合类型（如 `Vec<T>`、`String`、数组 `[T; N]`）转换为切片类型（`&[T]` 或 `&str`），从而调用切片类型的方法**，而无需用户手动写切片表达式（如 `&v[..]`）。
### **核心原则**：
- **减少显式借用**：允许编译器在方法调用上下文中自动生成对切片的借用。
- **保证所有权安全**：不移动原值，仅在已有借用的上下文下创建切片。
- **提升 ergonomics**：简化对集合类型调用切片方法的常见模式。
### 核心规则
**触发条件**：
- 发生在**方法调用中**。
- 目标类型是 `Vec<T>`、`String` 或数组 `[T; N]`。
- 方法接收 **`&[T]` 或 `&str` 类型的 self**。

**转换逻辑**：
```rust
Vec<T> -> &Vec<T> -> &[T]
[T; N] -> &[T]
```

对 `String`：

```rust
String -> &String -> &str
```
