## 1. 核心观点  

**异步 I/O（Asynchronous I/O）**：线程发起 I/O 请求后 **不阻塞当前线程**，可以继续执行其他任务。当 I/O 操作完成时，系统会通过 **回调、事件通知或信号**告知程序。

- 与 **阻塞 I/O** 相比：线程不会因等待 I/O 而挂起。**程序可以在 I/O 操作进行期间执行其他任务**。

> 核心思想：**I/O 操作和计算逻辑并行**，提升 CPU 利用率和程序吞吐量。

## 3. 展开说明  
### 本质
**都是暂停当前任务，让其他工作可以继续进行**

程并发系统中的阻塞式 I/O 和异步并发系统中的非阻塞式 I/O 是类似的。在这两种情况下，I/O 操作都需要时间，并且在 I/O 操作进行期间，其他工作也在同时进行：
- 使用线程时，执行 I/O 的线程会向操作系统请求 I/O，操作系统会暂停该线程，其他线程可以执行工作，当 I/O 完成后，操作系统会唤醒该线程，使其能够使用 I/O 的结果继续执行。
- 使用异步操作时，执行 I/O 的任务会向运行时请求 I/O，运行时再向操作系统请求 I/O，但操作系统会将控制权返回给运行时。运行时会暂停 I/O 任务，并调度其他任务来完成这些工作。当 I/O 操作完成后，运行时会唤醒 I/O 任务，使其能够使用 I/O 的结果继续执行。
