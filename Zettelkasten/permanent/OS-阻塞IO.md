## 1. 核心观点  

- **定义**：阻塞 I/O 是最直观的 I/O 模型。程序发起 I/O 请求后，**线程会被挂起，直到操作完成**，才能继续执行后续逻辑。
- **特点**：
	- **易于使用与实现**：逻辑顺序清晰，代码可读性高，顺序推理简单。
	- **顺序世界的限制**：程序必须等待当前 I/O 返回，无法同时处理其他任务。
    - **效率低**：线程在等待 I/O 完成时完全不能做其他工作，CPU 资源可能被浪费。

## 3. 展开说明  
- **使用场景**：
    - 简单脚本、命令行工具、小型程序，性能要求不高。
    - 教学或调试阶段，用于快速验证逻辑。
- **对比**：
    - 与非阻塞 I/O / 异步 I/O 相比，阻塞 I/O **实现简单但吞吐量低**。
- **底层价值点**：
	- 理解阻塞机制是掌握异步、事件驱动模型的前提。
    - 阻塞状态对应线程挂起，占用线程数据结构但不占用 CPU 指令周期。
