## 1. 核心观点  

#### 定义

`ManuallyDrop<T>` 是 Rust 标准库 `std::mem` 提供的一个封装类型，用于**阻止 Rust 自动调用值的析构函数（drop）**。它的定义大致如下：

`pub struct ManuallyDrop<T>(pub T);`

核心在于它实现了 `Copy` 或 `Deref` 的操作，但**没有实现 `Drop` trait**，因此被包裹的值不会在离开作用域时自动释放资源。

## 3. 展开说明  

#### 作用

1. **延迟析构**  
    当你希望控制某个值的销毁时机，而不是依赖作用域自动释放，可以用 `ManuallyDrop`。
    
2. **在 `union` 中安全使用非 `Copy` 类型**  
    Rust 中的 `union` 要求其字段不能有析构函数，否则编译器报错。用 `ManuallyDrop<T>` 包裹就可以绕开限制：

3. **实现自定义资源管理**  
	- 在自定义内存管理、异步运行时或者低层系统编程时，ManuallyDrop 常用于**手动释放资源**而避免重复析构或未定义行为。
