## 1. 核心观点  

在 Rust 中，你发现计算运算符（如 `+`, `-`, `*`, `/`）可以直接和引用（如 `&i32`）一起工作，这确实是一个非常实用的特性。

但最根本的原因依然是：**运算符是基于 Trait 的，而 Trait 为引用类型单独做了实现。**

## 3. 展开说明  
Rust 的运算符（如 `+`）对应 `Add` trait。为了让开发者写得爽，标准库为数字类型（如 `i32`）实现了所有可能的组合。

以 `i32` 为例，标准库里其实写了四种 `impl`：

- `impl Add<i32> for i32` （值 + 值）
    
- `impl Add<&i32> for i32` （值 + 引用）
    
- `impl Add<i32> for &i32` （引用 + 值）
    
- `impl Add<&i32> for &i32` （引用 + 引用）
    

这就解释了为什么你可以写 `5 + &10` 或者 `&5 + &10`，因为每一种情况都有对应的代码实现。`
