## 1. 核心观点  

`_` 是一种特殊的模式，**它匹配任何值并且不绑定到该值**。这告诉 Rust 我们不会使用该值，因此 Rust 不会警告我们未使用的变量。

这解决了什么问题：**对一些特定值执行特殊作，但对于所有其他值，执行一个默认操作,显式忽略了最后一个match中的所有其他值;**。

## 3. 展开说明  
### 简单示例
```rust
    let dice_roll = 9;
    match dice_roll {
        3 => add_fancy_hat(),
        7 => remove_fancy_hat(),
        _ => (),
    }

    fn add_fancy_hat() {}
    fn remove_fancy_hat() {}

```

此示例还**满足详尽性要求**，因为我们**显式忽略了最后一个match中的所有其他值;我们没有忘记任何东西**。
>这里表示我们不想运行任何代码。
