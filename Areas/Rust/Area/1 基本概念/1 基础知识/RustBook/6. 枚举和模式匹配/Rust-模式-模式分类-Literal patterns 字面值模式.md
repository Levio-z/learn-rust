## 1. 核心观点  

- 基本概念
	- **Literal Pattern**：模式匹配中直接写出的任何字面量值（，比如数字、字符、布尔值、字符串字面量等。
	- 匹配规则：**只有被匹配的值与文字完全相同，才会匹配成功**。
- **负数不是字面量**
    - 匹配负数时，使用 `-` 前缀，它是操作符，不是字面量
- **用途**
    - 适用于枚举、整数、字符等 **可精确比较的值**
    - 方便快速进行条件匹配

## 3. 展开说明  
### 简单示例
| 模式写法      | 匹配目标      | 说明                 |
| :-------- | :-------- | :----------------- |
| `5`       | `5`       | 精确匹配               |
| `true`    | `true`    | 精确匹配布尔值            |
| `-5`      | `-5`      | 匹配负数，实际是 `-` + `5` |
| `"hello"` | `"hello"` | 匹配字符串字面量           |
### 负数 **不是字面量**
Rust 中的负数 **不是字面量**（negative numbers are not literals）
- `-5` 实际上是操作符 `-` 作用于字面量 `5`
- 因此，模式中匹配负数时，需要加可选的减号 `-`，其语义类似于取反操作
