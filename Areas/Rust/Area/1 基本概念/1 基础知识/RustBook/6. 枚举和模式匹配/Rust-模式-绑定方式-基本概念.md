## 1. 核心观点  
为了更好地满足人体工程学要求，模式以不同的绑定方式运行，以便更轻松地绑定对值的引用。当一个**引用类型的值**被[非引用模式](Rust-模式-模式分类-Non-reference%20patterns非引用模式.md)（non-reference pattern）**匹配时，Rust 会自动把这个匹配视为 `ref` 或 `ref mut` 绑定。。例：
```rust
let x: &Option<i32> = &Some(3);
if let Some(y) = x {
    // y was converted to `ref y` and its type is &i32
}
```

默认绑定模式：
- 如果绑定模式没有显式具有 `ref`、`ref mut` 或 `mut`，则它使用_默认绑定模式_来确定变量的绑定方式。
- 默认绑定模式从使用移动语义的“移动”模式开始。

- 匹配模式时，编译器从模式的外部开始，向内工作。
	- 每次**使用非引用模式匹配引用**时，它都会自动取消引用该值并更新默认绑定模式。
	- 引用会将默认绑定模式设置为 `ref`。
	- 可变引用会将模式设置为 `ref mut`，除非该模式已经是 `ref`，在这种情况下它仍然是 `ref`。
	- 如果自动取消引用的值仍然是引用，则取消引用并重复此过程。
