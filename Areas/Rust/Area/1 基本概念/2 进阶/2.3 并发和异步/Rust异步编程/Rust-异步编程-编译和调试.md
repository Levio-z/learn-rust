## 3. 展开说明  

### 编译错误
异步 Rust 中的编译错误符合同步 Rust 的高标准，但由于异步 Rust 通常依赖更复杂的语言特性，如生命值和钉顶，因此**你可能会更频繁地遇到这类错误**。

### 运行时错误
每当编译器遇到异步函数时，它都会在底层生成状态机。异步 Rust 中的**栈跟踪通常包含这些状态机的详细信息，以及运行时的函数调用。因此，解析栈轨迹可能比同步 Rust 更复杂**。

### [新的失效模式](https://rust-lang.github.io/async-book/01_getting_started/03_state_of_async_rust.html#new-failure-modes)

异步 Rust 中可能出现一些新颖的失败模式，例如你从异步上下文调用阻断函数（blocking function），或者错误实现 `Future` 特性。这些错误可以悄无声息地通过编译器，有时甚至通过单元测试。对本书旨在传授的基本概念有扎实的理解，可以帮助你避免这些陷阱。
