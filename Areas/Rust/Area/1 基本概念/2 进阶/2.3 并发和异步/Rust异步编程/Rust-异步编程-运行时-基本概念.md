## 1. 核心观点  

- **性能考量：避免无用运行时占用内存和CPU。**
>原因：Rust 是一种底层语言，力求将运行时开销降至最低，
>- Java或Go即使不用异步也会加载运行时。只要你不用异步，rust编译出的代码没有运行时，不用就不付费原则。

- **灵活性：允许你根据自身需求选择合适的运行时环境**
>允许选择最适合的第三方运行时，不强制绑定特定实现方式。
>- 此外，设计和实现异步运行时环境的方法有很多种，，而不是提供一个现成的方案。这意味着，开始使用异步编程需要额外的步骤。

## 3. 展开说明  

### 职责
#### **异步任务必须进行管理和调度**。
- 通常情况下，任务数量会多于可用核心数，因此无法同时运行所有任务。当一个任务停止执行时，必须选择另一个任务来执行。如果一个任务正在等待 I/O 或其他事件，则不应调度它；但当等待完成时，则应调度该任务。**这需要与操作系统交互并管理 I/O 操作**。
#### 与操作系统交互以管理异步 I/O，它还必须为任务提供定时器功能（这与 I/O 管理密切相关）
- _反应堆_或_事件循环_或_驱动程序_ （等效术语）：分发 I/O 和定时器事件，与操作系统交互，并执行执行的最低级驱动。
- _调度器_ ：决定任务何时执行以及在哪些操作系统线程上执行。
- _执行器_或_运行时_ ：结合了反应器和调度器，是面向用户的用于运行异步任务的 API； _运行时_也指整个功能库（例如，Tokio crate 中的所有内容，而不仅仅是由 [`Runtime`](https://docs.rs/tokio/latest/tokio/runtime/struct.Runtime.html) 类型表示的 Tokio 执行器）。
- 除了上文所述的执行器之外，**运行时 crate 通常还包含许多实用特性和函数**。
	- 这些特性和函数可能包括用于 I/O 的特性（例如 `AsyncRead` ）和实现、用于常见 I/O 任务（例如网络连接或访问文件系统）、锁、通道和其他同步原语的功能、用于计时的实用工具、用于与操作系统交互的实用工具（例如信号处理）、用于处理 future 和流（异步迭代器）的实用函数，以及监控和观察工具。本指南将介绍其中的许多内容。
### 运行时选择
运行时具有截然**不同的调度策略**，有些则针对**特定任务或领域进行了优化**。本指南的大部分内容将使用 [Tokio](https://tokio.rs/) 运行时。它是一个通用运行时，也是该生态系统中最流行的运行时。对于入门和生产环境来说，它都是一个不错的选择。在某些情况下，使用其他运行时可能会获得**更好的性能**，或者能够编写**更简洁的代码**。本指南稍后将讨论其他一些可用的运行时，以及您可能选择某个运行时的原因，甚至包括编写您自己的运行时。
