- 在 Rust 中，所有项目（函数、方法、结构、枚举、 模块和常量）默认对父模块是私有的。如果你想**为了使函数或结构等项目成为私有的，可以将其放入模块中。**
- 父模块中的项目不能使用子模块中的私有项目，但子模块中的项目可以使用其祖先模块中的项目。**这是因为子模块包装并隐藏了它们的实现细节，但子模块可以看到它们定义的上下文。**

|   |   |
|---|---|
|**子 → 父/祖先**|子模块**可以**访问祖先模块的所有项（包括私有项）|
```rust
fn private_function() {

    // 私有项之一

}

  

mod child {

    pub fn fix_incorrect_order() {

        // ✅ 父模块的私有项对子模块是可见的

        super::child::public_function(); // ✅ 父模块子模块

        super::private_function(); // ✅ 父模块私有项之一

  

        // ✅ 同模块不分公有和私有

        child2::public_function(); // ✅ 允许访问子模块公有项

        child2::priate_function(); // ❌ 不允许访问子模块私有项

        // ✅ 允许访问crate根模块中所有项

        crate::child::public_function(); // ✅ 允许访问crate根模块中所有项

    }

    mod child2 {

        // 子模块公有项

        pub fn public_function() {}

        // 子模块私有项

        fn priate_function() {}

    }

}
```
