## 1. 核心观点  
### Ⅰ. 概念层

**__注册中心_是一种包含可下载文件集合的服务。 可以安装或用作依赖项的 [_crate_](https://doc.rust-lang.org/cargo/appendix/glossary.html#crate) [_软件包_](https://doc.rust-lang.org/cargo/appendix/glossary.html#package) 。Rust 生态系统中的默认注册表是 [crates.io](https://crates.io/) 。该注册表有一个[_索引_](https://doc.rust-lang.org/cargo/appendix/glossary.html#index) ，其中包含所有 crate 的列表，并告诉 Cargo 如何下载所需的 crate。

## 3. 展开说明  
**可发现性** 。中央注册表提供了一个便捷的查找现有软件包的平台。结合标签功能，注册表还可以提供生态系统范围内的信息，例如最受欢迎或依赖性最高的软件包列表。
**速度** 。中央注册表使得快速高效地获取软件包的元数据成为可能，然后仅高效地下载已发布的软件包，而无需下载仓库中其他冗余内容。这显著提高了依赖关系解析和获取的速度。随着依赖关系图的规模扩大，下载所有 Git 仓库会迅速导致速度下降。此外，请记住，并非每个人都拥有高速、低延迟的互联网连接。
