## 3. 展开说明  
如您所知，在 Rust 中，crate或可执行程序被称为库或可执行程序。
使用 Rust 编译器进行编译 `rustc`。首先会遇到 rustc 源代码。 就是经典的“Hello World”程序，他们通过调用以下函数来编译它： `rustc` 直接：
```
$ rustc hello.rs 
$ ./hello 
Hello, world!
```
请注意，上述命令要求您显式指定文件名。如果您要直接使用 `rustc` 编译其他程序，则需要不同的命令行调用。如果您需要指定任何特定的**编译器标志**或包含**外部依赖项**，则所需的命令会更加具体（也**更加复杂**）。

此外，大多数非平凡程序很可能依赖于外部库，因此也会间接地依赖于_它们的外部_库。 依赖项。获取所有必要依赖项的正确版本。 如果由……来做，要**手动保持**它们的最新状态将**既困难又容易出错**。

与其仅仅使用 crates 和 `rustc` ，不如避免这些困难。 通过引入更高层次的方法来参与执行上述任务 [“ _包_ ”](https://doc.rust-lang.org/cargo/appendix/glossary.html#package "\"package\" (glossary entry)") 抽象，并通过使用 [_软件包管理器_](https://doc.rust-lang.org/cargo/appendix/glossary.html#package-manager "\"package manager\" (glossary entry)") 。
