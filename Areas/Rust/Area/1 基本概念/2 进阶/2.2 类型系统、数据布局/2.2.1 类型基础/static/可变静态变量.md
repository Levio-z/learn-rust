#### 1. 不可变静态变量 `static`

`static BUFFER_SIZE: usize = 1024;`
- 编译器保证它在整个程序生命周期中只存在一份。
- **默认不可变，因此只能通过不可变引用 `&` 访问。**
	- 生命周期 `'static` 保证访问有效性。
	- 不可变性保证线程安全。
	- 可变性需要内部可变性或同步机制来保证线程安全。
- **线程安全**：访问时无需额外同步机制。
- **限制**：不能调用需要 `&mut self` 的方法，因为它是不可变的。

#### 2. 可变静态变量 `static mut`
```
static mut COUNTER: u32 = 0;
```
可以被修改，但访问必须使用 `unsafe` 块：
```
unsafe {
    COUNTER += 1;
}
```
- Rust 之所以这样设计，是为了防止数据竞争。因为静态变量在全局唯一，如果多线程同时访问且没有同步机制，会引发**未定义行为**。
- 本质上是存在“可变静态变量”，但它不安全且受限。
