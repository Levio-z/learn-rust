## 1. 核心观点  
### Ⅰ. 概念层

`rustyline` 是一个 Rust 语言实现的 readline 风格库，用于构建交互式命令行接口（REPL、shell、命令工具等）。它提供了行编辑、历史记录、自动补全、语法高亮和键绑定等功能，类似于 Unix 系统上的 GNU Readline，但完全用 Rust 编写，因此无需依赖 C 库。

主要用途：

- **交互式命令行输入**：允许用户在终端输入、编辑和执行命令。
- **历史记录管理**：保存用户输入，支持上下键快速浏览历史命令。
- **自动补全**：可自定义补全逻辑，提升用户体验。
- **键绑定自定义**：支持 Vi 模式或 Emacs 模式的键绑定。
- **可嵌入 REPL/CLI 工具**：无需依赖外部 readline 库，跨平台兼容性好。

## 3. 展开说明  
### 核心特性

1. **行编辑（Line Editing）**  
    支持光标移动、文本删除、单词跳转、撤销/重做等操作。
    
2. **历史管理（History）**
    - 内存中保存历史
    - 可持久化到文件
    - 支持搜索历史命令
        
3. **自动补全（Completion）**
    - 实现 `Completer` trait
    - 可以对路径、命令或自定义词表进行补全
    - 可控制补全行为（如循环补全、补全忽略大小写）
        
4. **提示符与高亮（Prompt & Highlighting）**
    - 自定义提示符文本和颜色
    - 高亮输入行特定部分（可用于语法提示）
5. **多种模式支持（Keybindings）**
    - Emacs 模式：类似 GNU Readline 默认行为
    - Vi 模式：命令/插入模式切换
6. **跨平台兼容**
    - Linux / macOS / Windows（cmd.exe, PowerShell）
    - 不依赖外部 C 库
