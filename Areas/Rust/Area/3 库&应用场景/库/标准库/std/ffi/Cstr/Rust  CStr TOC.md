### 1. 总结
- **作用**：将字节切片安全地视作 C 字符串
- **原理**：检查末尾 null 并禁止内部 null
- **使用场景**：
    - Rust ↔ C FFI
    - 解析二进制协议或文件中的 C 风格字符串
- **底层价值点**：
    - 区分 Rust 字符串和 C 字符串
    - 了解 null 字节在字符串中的语义
    - 零拷贝安全转换
### 2. 概念
- **与 C 接口交互**
    - C 语言的字符串是**以 `\0` 结尾的字节序列 (`char*`)**。
    - Rust 的 `&str` 是 UTF-8 串，**不保证 null 结尾**。
    - `CStr` 提供安全的方式在 Rust 中引用 C 字符串：
        - 不会在 Rust 里修改 C 字符串（**不可变借用**）。
        - 提供转换为 Rust 字符串的方法。
- **安全转换**
    - 可以用 `CStr` 来读取 `*const c_char` 或 `[u8]`。
