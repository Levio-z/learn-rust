## 3. 展开说明  
### 线程

**操作系统线程**不需要对编程模型做任何更改，这使得表达并发变得非常容易。然而，线程间的同步可能较为困难，且性能开销较大。线程池可以减轻部分成本，但不足以支持庞大的 IO 负载。

- [线程实现并发的方式](../../../../../../../Zettelkasten/permanent/线程实现并发的方式.md)
- [线程并发的缺点](../../../../../../../Zettelkasten/permanent/线程并发的缺点.md)
### **Event-driven programming**

事件**驱动编程**结合回调 ，**性能非常高，但往往会导致冗长且“非线性”的控制流**。数据流和误差传播往往难以追踪。
- [操作系统-并发模型-事件驱动编程](操作系统-并发模型-事件驱动编程.md)
### 协程
**协程**和线程一样，不需要对编程模型做修改，这让它们易于使用。像异步一样，它们也能支持大量任务。然而，它们会抽象出对系统编程和自定义运行时实现者重要的底层细节。
- [操作系统-任务的协作调度-异步编程中的协作式调度](../../../../../../../Zettelkasten/fleeting/操作系统-任务的协作调度-异步编程中的协作式调度.md)
