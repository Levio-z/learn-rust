协议（Protocol）就是一套**公认、详细、必须遵守**的“**规则** (制性强制的、必须遵守的)和**约定**(推荐的、默认的、共识性的)”，用于确保两个或多个实体（例如客户端和服务器、不同国家的人）能够成功地进行**交流**、理解和协作。
- 因为：计算机与网络设备要相互通信，双方就必须基于相同的方法
### 详解“协议作为规范”

#### 1. 统一的语言和格式

在 Web 世界中，协议（HTTP）定义了通信双方**如何“说话”**：

- **格式（Syntax）：** 客户端发送的请求报文应该长什么样？服务器返回的响应报文应该长什么样？（例如：请求必须包含一个方法、一个 URL 和 HTTP 版本号，然后是头部字段，最后是可选的正文。）
- **动作（Semantics）：** 客户端能做什么？（例如：`GET` 是获取资源，`POST` 是提交数据。）服务器对这些动作应该如何回应？（例如：`200 OK` 表示成功，`404 Not Found` 表示资源不存在。）
- **顺序（Sequence）：** 通信的步骤是什么？（例如：先建立 TCP 连接，然后客户端发送请求，然后服务器发送响应，最后关闭连接或保持连接。）
    

#### 2. 互操作性（Interoperability）的基础
- **没有协议：** 如果没有 HTTP 协议，每个浏览器和服务器都可以用自己喜欢的方式发送数据。A 浏览器发送的数据，B 服务器可能完全看不懂，反之亦然。通信就会失败。
- **有了协议：** 协议就像一种通用的翻译工具。只要一个软件声称自己支持 HTTP 协议，它就必须遵循这套规范。这意味着：
    - **Google Chrome**（客户端）知道如何构造一个标准的 HTTP 请求。
    - **Apache/Nginx**（服务器）知道如何解析这个请求，并构造一个标准的 HTTP 响应。

因此，**HTTP 规范**确保了不同公司、不同编程语言、不同操作系统开发的客户端和服务器能够**无障碍地互相理解和通信**，实现了全球 Web 的互联互通。

#### 3. 规范的约束力

协议一旦被确立为规范，它就具有**约束力**。任何**想要参与** Web 通信的软件（浏览器、服务器、爬虫、API 客户端等）都必须遵守 HTTP **规范中定义的全部规则**。如果你不遵守，你的软件就无法正常地进行 Web 通信。
