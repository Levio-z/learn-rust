### POST 方法的核心作用：传输实体主体
- **POST** 方法用于向 URI 标识的资源提交（Submit）数据，这些数据通常包含在请求的**实体主体 (Entity Body)** 中。
- 服务器通常会将这些数据视为对请求 URI 所标识资源的**附加、修改或处理**。
### 常见的应用场景：
1. **提交表单数据：** 这是最常见的用途。当用户在网页表单中输入数据并点击提交时，数据通常通过 POST 请求，放在实体主体中发送给服务器。
2. **创建新资源：** POST 通常用于指示服务器**创建一个新的子资源**。例如，向 `/users` URI 发送 POST 请求，其主体包含新用户的信息，服务器在数据库中创建该用户。
3. **发送大量数据：** 由于 GET 方法将数据放在 URI 的 Query 部分，受到 URI 长度限制，而 POST 方法将数据放在实体主体中，因此适合传输大量或复杂的结构化数据（如 JSON、XML）。
4. **执行操作：** 虽然 RESTful 设计推荐使用 PUT/PATCH 进行更新，但 POST 也可以用来触发服务器上的一些非幂等操作。

### POST 方法的重要特性（与 GET 的区别）：
- **非幂等性 (Non-Idempotent):** 多次重复提交同一个 POST 请求，通常会产生**多次效果**（例如，多次创建相同的资源或重复处理事务）。
- **不安全 (Unsafe):** POST 方法会改变服务器上的资源状态，因此被认为是非安全的。
- **必需实体主体：** 虽然技术上允许空的请求体，但 POST 的设计目的就是为了携带数据，所以通常包含实体主体。
