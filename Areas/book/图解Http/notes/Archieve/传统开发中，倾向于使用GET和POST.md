是当页面通过标准 HTML 表单进行交互时，**使用 `POST` 来模拟 `PUT`（更新）和 `DELETE`（删除）** 是一种非常常见的模式。

这种模式之所以流行，主要基于您提到的两个原因：
### 1. 浏览器的原生限制
[表单只允许两种 `method`](表单只允许两种%20`method`.md)
### 2. 简化安全和统一处理
如前所述，**`POST` 请求总是会命中服务器端的应用程序脚本，使得开发者可以将所有的安全、验证、授权逻辑集中在这些处理脚本中**。通过使用 `POST` 模拟，开发者可以：
- **统一认证/授权流程：** 无论是更新还是删除，都走同一套 `POST` 处理流程，确保用户已登录且有权限。
- **兼容性：** 保证所有用户和所有浏览器都能顺利执行操作。
	- 浏览器的原生功能兼容性
		- **所有浏览器、所有版本**都能正确地提交表单。
		- **不需要依赖 JavaScript** 或特定的前端库来发送高级请求，这对于不支持 JavaScript 或禁用了脚本的用户，或对于传统的、非 SPA（单页应用）网站非常重要。
	- 核心原因：GET 和 POST 是 HTTP 协议中最早、最基础、使用最广泛的方法，几乎所有网络基础设施都默认且正确地支持它们。
