**目的：请求访问资源**
- GET方法用来请求访问已被URI识别的资源。
- 服务器处理：解析与响应
	- **静态资源：** 如果 URI 标识的是一个**静态文件**（如 HTML 文本、图片、CSS 文件等），服务器会直接读取该文件并将其内容作为响应报文的主体返回。
	    - “如果请求的资源是文本，那就保持原样返回”_
	- **动态资源：** 如果 URI 标识的是一个**程序**（如 CGI 脚本、PHP/Python/Java 程序等），服务器会执行这个程序。程序运行后产生的**输出结果**（通常是动态生成的 HTML 或 JSON/XML 数据）会被捕获，并作为响应报文的主体返回给客户端。
	    - _“如果是像 [HTTP-CGI](HTTP-CGI.md)那样的程序，则返回经过执行后的输出结果”_
