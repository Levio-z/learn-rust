**SSH** 的全称是 **Secure Shell**（安全外壳协议）。它是一种加密的网络传输协议，目的是在不安全的网络环境中，为网络服务提供**安全的**传输环境。

简单来说，SSH 就是用来**安全地连接到远程计算机（通常是服务器）**，并在上面执行命令、传输文件的工具和协议。

### 核心功能与特点

1. **加密（Encryption）：**
    - 这是 SSH 最核心的功能。与早期的 Telnet、FTP 协议不同，SSH 会对客户端和服务器之间传输的所有数据进行**加密**。
    - 这意味着即使数据被第三方截获（例如通过网络监听），攻击者也无法读取或理解其中的内容（包括您的密码或传输的文件内容）。
2. **身份验证（Authentication）：**
    
    - SSH 提供强大的身份验证机制，用于验证客户端和服务器的身份，确保您连接到的是正确的服务器，并且是授权用户在使用。
        
    - 最常见的认证方式是：
        
        - **密码认证：** 使用用户名和密码登录（虽然安全，但不如密钥安全）。
            
        - **密钥对认证（Key-Pair Authentication）：** 使用公钥（Public Key）和私钥（Private Key）进行认证，这是**最安全且最推荐**的方式，尤其是在 Git 和服务器管理中。
            
3. **远程控制和会话：**
    
    - SSH 最常见的用途是远程登录到服务器，让您可以在本地终端上像操作本地计算机一样，在远程服务器上执行命令行操作。
        

### SSH 在 Git 中的作用

在 Git 中，当您使用 `ssh://` 协议克隆、拉取或推送代码时，SSH 扮演了至关重要的安全角色：

1. **安全传输：** 保证您和 Git 仓库服务器之间的数据（代码、提交信息等）传输是完全加密的。
    
2. **密钥认证（最佳实践）：** Git 通常使用 SSH 的**密钥对认证**。您将公钥添加到 GitHub/GitLab 等服务账户中，使用私钥进行认证。
    
    - **优点：** 一旦设置好，您在 Git 操作时通常**无需输入密码**，因为它会自动使用私钥认证，既安全又方便。
