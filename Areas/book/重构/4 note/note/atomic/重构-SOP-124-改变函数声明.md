## 1. 核心观点  
### Ⅰ. 概念层

**变函数声明**是一种重构手法，用于**修改函数的名称、参数列表或返回形式**，以更准确地表达函数职责、降低误用风险并提升可读性与可维护性。常见形式包括：重命名函数、增删参数、参数重排、引入参数对象、拆分/合并返回值。

## 3. 展开说明  

### 初始设置｜动机与作用

- **语义对齐**：名称与参数应直接反映真实业务含义
- **降低耦合**：避免调用方依赖隐含规则（如参数顺序、魔法值）
- **提升演进性**：为后续功能扩展提供稳定接口
- **减少错误**：通过类型与结构约束减少误用

### 初始设置｜适用场景

- 函数名**模糊或误导**（如 `process()`、`handle()`）
- 参数**过多或含义不清**（布尔旗标、位置敏感）
- 参数**经常一起出现**（可抽为参数对象）
- 返回值**复合含义**（可返回结构体/对象）
- 调用点**频繁改动**，接口演进受阻

### 初始设置｜重构形式

1. **重命名函数**：强调“做什么”，而非“怎么做”
2. **增删参数**：显式化必要信息，移除冗余
3. **参数重排**：按重要性/稳定性排序
4. **引入参数对象**：聚合强相关参数
5. **拆分/合并返回值**：让返回语义自解释

### 初始设置｜操作步骤（安全路径）

1. **建立覆盖测试**（确保行为不变）
2. **引入新声明**（并行存在，保持兼容）
3. **迁移调用点**（逐个替换）
4. **删除旧声明**（清理技术债）

> 小步提交，保持可回滚性。


### 初始设置｜原理要点

- **接口即契约**：函数声明是调用方与实现的边界
- **稳定性优先**：先扩展、后收敛，避免破坏性变更
- **类型即文档**：用类型系统表达约束与意图
- **最少惊讶原则**：调用者无需阅读实现即可正确使用

### 初始设置｜与相关手法的关系

- 与 **以查询取代临时变量**：接口清晰后更易抽取查询
- 与 **引入参数对象**：解决参数爆炸
- 与 **搬移函数**：声明清晰后更易判断归属
- 与 **封装条件/多态替代条件**：减少布尔参数
