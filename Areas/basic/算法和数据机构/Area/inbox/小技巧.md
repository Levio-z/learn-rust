`Vec::new()` 创建的是一个**空向量**，长度为 `0`，容量为 `0`，这时里面没有任何元素，不能用索引直接写入，否则会越界。
如果你想往里面填元素，必须使用 `push()` 或者 `resize()`：
```rust
let mut v = Vec::new();
v.resize(n, 0);  // 用0扩展到长度n
```

### 有时候需要使用let v = vec!`[0; 5]`;初始化
`vec![expr; n]` 是 Rust 的**宏**语法糖，等价于：
```rust
let mut v = Vec::with_capacity(n);
v.resize(n, expr);
```
也就是说，它创建了一个长度为 `n` 的向量，并用 `expr` 的拷贝值填充。  
例：
```rust
let v = vec![0; 5]; // [0, 0, 0, 0, 0]
```
知道新数组的长度，并且可以从数组末尾初始化新数组，没有初始化，`ans[p]` 会直接触发 panic（越界访问）

### 为什么Vec::with_capacity(n)不行
-   `Vec::with_capacity(n)`只是预留了容量，但长度为0
- 因此它不能直接通过索引访问，也不能直接写入：
```rust
let mut v = Vec::with_capacity(5);
v[0] = 10; // ❌ panic: index out of bounds
```
