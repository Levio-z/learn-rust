```
gdbserver: build
	@qemu-system-riscv64 -machine virt -nographic -bios $(BOOTLOADER) -device loader,file=$(KERNEL_BIN),addr=$(KERNEL_ENTRY_PA) -s -S

```

这段 Makefile 代码定义了一个用于 **在 QEMU 上启动并调试 RISC-V 裸机内核** 的目标。我们逐项进行**逐行详解**：
- 在 QEMU 中加载裸机内核镜像、挂载 bootloader 并启动 GDB 远程调试服务器。
     `gdbserver: build`
    - `gdbserver` 是 make 的目标名。
    - 依赖 `build` 目标：即在执行 `gdbserver` 前，先执行 `build`，通常用于编译 `KERNEL_ELF` 与生成 `KERNEL_BIN`。
    `s`
    - 启用 GDB 远程调试功能（`gdb tcp::1234`）。
    - QEMU 会开启一个 TCP 端口（默认 1234），等待 GDB 客户端连接。
    `S`
    - 启动后暂停 CPU，直到 GDB 发出继续命令。
    - 用于在程序启动前进行调试，如设置断点、检查寄存器等。
