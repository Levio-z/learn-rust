### 首次提交信息
```
chore(init): initial commit
```

在开发中，团队除了 `TODO:` 和 `FIXME:`，还经常使用一些其他注释标记，用于提醒、说明或记录特殊情况。常见有：

| 标记            | 作用                  | 示例                                                        |
| ------------- | ------------------- | --------------------------------------------------------- |
| `NOTE:`       | 提示或备注重要信息，但不是错误或待办  | `// NOTE: This function assumes input is sanitized`       |
| `HACK:`       | 临时解决方案或不优雅实现，需要将来改进 | `// HACK: Quick fix, replace with proper algorithm later` |
| `REVIEW:`     | 需要代码审查或复查           | `// REVIEW: Check performance of this loop`               |
| `OPTIMIZE:`   | 有性能优化空间             | `// OPTIMIZE: Could reduce memory allocation here`        |
| `BUG:`        | 明确已知 bug            | `// BUG: Crashes when input array is empty`               |
| `IDEA:`       | 记录想法或改进建议           | `// IDEA: Consider caching result for faster lookup`      |
| `QUESTION:`   | 提出问题或疑问             | `// QUESTION: Why does this branch never execute?`        |
| `DEPRECATED:` | 标记过时代码              | `// DEPRECATED: Use new_parse() instead`                  |
