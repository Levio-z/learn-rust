## 1. 核心观点  

SDK 是**代码组件**，通过封装复杂逻辑，供应用调用；而服务是**独立运行的系统组件**，提供具体功能或资源。

- **本质**
    - SDK 是一组**库、工具、接口和文档**，供开发者在应用中调用。
    - 它本身不是独立运行的服务，而是集成到客户端应用中的代码组件。
        
- **与服务的区别**

|特性|SDK|服务（Server）|
|---|---|---|
|运行方式|集成到应用中调用|独立进程或系统提供服务|
|功能定位|封装逻辑、提供接口|提供业务功能或资源访问|
|控制权|开发者应用控制|服务端控制，客户端访问|
|举例|消息队列 SDK、数据库 SDK|Kafka Broker、MySQL Server|
**在你的场景中的角色**

- SDK 封装了**轮询、主备切换、容错逻辑**。
    
- 客户端应用调用 SDK 的接口来发送消息。
    
- SDK 并不提供消息存储或网络服务本身，这些由消息队列服务器承担。
