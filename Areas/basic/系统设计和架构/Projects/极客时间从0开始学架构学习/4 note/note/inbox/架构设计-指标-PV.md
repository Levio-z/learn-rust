## 1. 核心观点  
### Ⅰ. 概念层

PV 是**页面级访问事件计数**，表示页面被成功加载一次。

**作用**
- 业务活跃度衡量（内容型产品核心指标）
- 容量估算的中间变量（DAU → PV → QPS）

****本质**
- **用户行为事件**，不是系统调用
- 粒度：页面 / 视图

**使用场景**
- 评估内容消费强度
- 估算总体访问规模

## 3. 展开说明  
### QPS与PV的关系

QPS≈PV×每 PV 的平均请求数

| 架构形态   | PV → 请求数       |
| ------ | -------------- |
| 传统 MPA | 1 PV ≈ 1 请求    |
| SPA    | 1 PV ≈ N 个 API |
| BFF    | N API → 1~2 请求 |
| 延迟加载   | 请求数 > PV       |
**工程要点**
- **请求数由接口设计决定**
- BFF/聚合接口可显著降低 QPS
