```
- ./init.sql:/docker-entrypoint-initdb.d/init.sql:ro  # ✅ 初始化脚本只读挂载
```

| 部分                                     | 含义                                   |
| -------------------------------------- | ------------------------------------ |
| `./init.sql`                           | 宿主机当前目录下的 `init.sql` 文件              |
| `/docker-entrypoint-initdb.d/init.sql` | 容器中的路径，MySQL 容器会自动执行这个目录下的 `.sql` 脚本 |
| `:ro`                                  | 表示只读挂载（Read-Only）                    |

MySQL 官方镜像具有这样的机制：

> **容器首次启动时**，如果 `/var/lib/mysql` 数据目录为空（即数据库还没初始化），那么：
> 
> 容器会自动执行 `/docker-entrypoint-initdb.d/` 目录下的：
> 
> - 所有的 `.sql` 文件（如：`init.sql`）
>     
> - 所有的 `.sh` 脚本
>     
> - 所有的 `.sql.gz` 文件（自动解压执行）
>
