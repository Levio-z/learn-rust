## 1. 核心观点  
### Ⅰ. 概念层

在 Rust 中，**指针封装类型（Pointer Wrapper / Pointer-Like Type）**指的是一种“包装了指针但增加额外语义或限制”的类型，它本质上仍然指向某块内存，但对访问或操作方式做了约束或增强。
### 核心特征

1. **内部持有指针**
    - 这类类型内部通常包含一个裸指针或智能指针，例如 `*mut T`、`Box<T>`、`Arc<T>` 等。
    - 它本身并不是数据，而是“指向数据”的引用。
2. **增加额外语义或约束**
    - 比普通指针多了类型系统或 API 层面的保证：
        - `Pin<P>`：保证指向的数据不会被移动。
        - `Box<T>`：保证堆分配、独占所有权。
        - `Rc<T>` / `Arc<T>`：允许多所有者引用计数管理。
    - 这些封装类型提供了对底层指针的安全访问方式。
3. **不能直接移动或复制内部指针**
    - 一些封装类型（如 `Pin`）限制对内部指针的移动或获取可变引用，确保内存安全。
    - 封装类型本身是 Rust 类型系统的一部分，可以在编译时检查操作是否合法。
