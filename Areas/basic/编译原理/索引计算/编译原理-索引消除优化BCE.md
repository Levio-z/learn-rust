## 1. 核心观点  
- Rust 数组和 `Vec` 的访问是 **安全的**，每次 `nums[i]` 都会检查 `i < nums.len()`，超界会 panic。
- **Bounds Check Elimination（BCE）** 是编译器优化的一种：如果能 **静态证明索引永远合法**，就会去掉边界检查。
```rust
let nums = vec![1, 2, 3, 4];
for i in 0..nums.len() {
    let x = nums[i]; // 编译器可消除边界检查
}
```
因为 `i` 总在 `0..nums.len()` 范围内，LLVM 可以安全移除检查。
