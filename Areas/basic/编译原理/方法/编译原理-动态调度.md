## 3. 展开说明  
### 对象

面向对象系统将问题建模为一组相互作用的对象，这些对象执行由名称指代的操作。多态性是指某些可互换的对象各自公开一个同名操作，但行为可能不同的现象。例如， File 对象和 Database 对象都具有一个 StoreRecord 方法，可用于将人事记录写入存储。它们的实现方式不同。程序持有一个对象的引用，该对象可能是 File 对象或 Database 对象。具体是哪个对象可能由运行时设置决定，而此时程序可能并不知道或关心是哪个。当程序调用对象的 StoreRecord 方法时，需要某种机制来选择执行哪种行为。如果将面向对象编程 (OOP) 理解为向对象[发送消息](https://en.wikipedia.org/wiki/Message_passing "Message passing") ，那么在这个例子中，程序向一个未知类型的对象发送 StoreRecord 消息，并将消息分发给正确对象的任务留给运行时支持系统。对象执行它所实现的行为。 [[ 2 ]](https://en.wikipedia.org/wiki/Dynamic_dispatch#cite_note-Driesen-Hoelzle-Vitek_1995-2)

动态分派与_[静态分派](https://en.wikipedia.org/wiki/Static_dispatch "Static dispatch")_形成对比，静态分派在[编译时](https://en.wikipedia.org/wiki/Compile_time "Compile time")选择多态操作的实现方式。动态分派的目的是推迟选择合适的实现方式，直到确定参数（或多个参数）的运行时类型。

动态分发与[后期绑定](https://en.wikipedia.org/wiki/Late_binding "Late binding") （也称动态绑定）不同。 [名称绑定](https://en.wikipedia.org/wiki/Name_binding "Name binding")将一个名称与一个操作关联起来。一个多态操作可以有多个实现，所有实现都与同一个名称关联。绑定可以在编译时进行，也可以（使用后期绑定）在运行时进行。而动态分发则是在运行时选择操作的某个特定实现。虽然动态分发并不意味着后期绑定，但后期绑定却意味着动态分发，因为后期绑定操作的具体实现直到运行时才确定。 [ _[需要引用](https://en.wikipedia.org/wiki/Wikipedia:Citation_needed "Wikipedia:Citation needed")_
