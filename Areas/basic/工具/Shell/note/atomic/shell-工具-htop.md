## 1. 核心观点  
### Ⅰ. 概念层

htop 是一个 **交互式进程监控工具**，用于实时查看 Linux/Unix 系统中的 **CPU、内存、进程状态与资源占用情况**。  
它是传统 `top` 命令的增强版本，提供更直观的界面、更丰富的交互能力和更高的可用性。

### Ⅱ. 应用层

- **实时监控系统负载**：CPU 使用率、内存/Swap 使用情况
- **进程管理**：查看、排序、过滤、结束进程
- **多核可视化**：每个 CPU 核心独立显示
- **快速定位问题进程**：如 CPU 飙升、内存泄漏、僵尸进程

## 3. 展开说明  
### 三、htop 的界面组成

- **顶部状态栏**
    - CPU：按核心显示使用率（不同颜色表示 user/system/iowait 等）
    - Memory / Swap：当前内存与交换区使用情况
- **中部进程列表**
    - PID、用户、优先级、CPU%、MEM%、运行状态、命令
- **底部快捷键提示**
    - F1–F10 功能键，支持直接操作
### 四、htop 的核心特性（对比 top）

- **支持方向键操作**（无需记复杂命令）
    
- **彩色 UI**，信息层次清晰
    
- **树状进程视图**（显示父子进程关系）
    
- **动态排序**（CPU / MEM / TIME 等）
    
- **内置过滤器**（按进程名、用户筛选）
### 五、常用快捷键

- `F2`：设置（显示列、颜色、排序方式）
    
- `F3`：搜索进程
    
- `F4`：过滤进程
    
- `F5`：树形视图
    
- `F6`：选择排序字段
    
- `F9`：Kill 进程（选择信号）
    
- `F10`：退出
