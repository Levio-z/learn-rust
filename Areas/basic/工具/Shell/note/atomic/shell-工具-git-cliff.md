## 1. 核心观点  
### Ⅰ. 概念层
- 核心功能：自动化生成符合规范的CHANGELOG.md文件
- 实用价值：
    - 帮助使用者快速了解各版本变更内容
    - 遵循conventional commits规范
- 典型输出：
- 安装命令：

```
cargo install git-cliff
```


### Ⅱ. 应用层

- [shell-工具-git-cliff-配置文件详解](../misc/shell-工具-git-cliff-配置文件详解.md)

```
# 生成完整的变更日志
git cliff

# 预览生成结果（不写入文件）
git cliff --dry-run

# 生成特定版本范围的变更日志
git cliff v1.0.0..v2.0.0

# 使用自定义配置文件
git cliff --config cliff.toml
```
