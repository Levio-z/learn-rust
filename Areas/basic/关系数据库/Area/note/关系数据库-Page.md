## 1. 核心观点  

1. **定义**
    - 数据库内部把存储分为固定大小的数据块，每块称为 **页（Page / Data Page）**
    - InnoDB 默认页大小 16KB
    - 一页里通常存若干行数据
2. **作用**
    - **最小 I/O 单位**：磁盘读写操作总是以页为单位
    - **Buffer Pool 缓存单位**：InnoDB 将页缓存到内存中处理事务
    - **事务一致性管理单位**：MVCC Undo / Redo 通常按页组织
3. **特性**
    - 页内行存储按行格式排列（Row Format: Compact / Redundant / DYNAMIC / COMPRESSED）
    - 访问某列 → 整行数据被加载到页缓冲（buffer pool）

> 注意：这是 **数据库逻辑页**，大小、组织方式由数据库设计，而非操作系统规定。

## 3. 展开说明  

#### 分区表作用

1. **提升查询性能**
    
    - 对分区键进行查询时，数据库可以进行“分区裁剪（Partition Pruning）”，**仅访问相关分区，显著减少 I/O。**
        
    - 典型场景如时间、状态等分布极不均的数据（例如 95% 订单为“完成”，5% 为“未完成”）。
        
2. **提高写入与维护效率**
    
    - 新增数据时直接落入目标分区；
	    - 直接定位目标分区，局部聚簇索引插入，不分区需在整张聚簇索引中查找插入位置
        
    - 删除过期数据时可直接 `DROP PARTITION` 而无需逐行删除。

	- 处理比如一个分区的数据，不需要全表扫描。
3. **提升可扩展性与并行性**
    
    - 不同分区可分布在不同物理磁盘或节点；
        
    - 可实现分区级**并行扫描、备份、恢复**。
        
4. **隔离冷热数据**
    
    - 热数据放 SSD，冷数据放 HDD，提升整体性价比。
