>根域的 DNS 服务器信息保存在互联网中所有的 DNS 服务器中。
- **4.1 客户端预检查：** 浏览器缓存 → 操作系统缓存 → `hosts` 文件。
- **4.2 递归查询 (Recursive Query)：** 客户端向**本地 DNS 服务器**发送的请求，客户端只需问一次。本地 DNS 必须给出最终答案。
	-  **发送方（客户端）：** 发出请求后，进入被动等待状态，不再参与后续的查询步骤。
	- **接收方（本地 DNS 服务器）：** 承担全部责任。它必须代表客户端去联系其他所有需要的服务器（根、TLD、权威服务器），直到获取到最终的 IP 地址。

>**递归请求**是指发送查询的一方（客户端）要求接收查询的一方（本地 DNS 服务器）**必须**完成整个解析过程，并返回**最终结果（即 IP 地址）**，而不是仅仅返回下一步应该询问谁的地址。**类似于算法中的递归调用：** 当一个函数被调用时，它被要求**全权负责**直到返回最终结果。如果它需要其他资源，它会在**内部**（在客户端不知道的情况下）自行进行多次调用（即迭代查询），最终将结果返回给发起者。

- **4.3 迭代查询 (Iterative Query)：** 本地 DNS 服务器向根、TLD、权威服务器之间发起的请求。每一步只给出下一跳的地址。本地 DNS 需要多次询问，直到找到答案。

>接收方**只返回它知道的最佳下一跳地址**，发送方（本地 DNS）必须自己重新发起新的查询。**类似于算法中的循环迭代：** 每一个被查询的服务器都只返回**最好的、下一步的提示**（即下一个要查询的服务器地址）。本地 DNS 必须根据这个提示，**主动地、重复地**发起新的查询（进入下一个“循环”）。它没有将责任委托出去，而是将责任保留在自己手中，通过**多次、循环的步骤**解决问题。

- **4.4 完整的查询流程 (例如查询 `www.server.com`)：**
    1. 客户端 → 本地 DNS (递归请求 `www.server.com`)。
    2. 本地 DNS → 根 → 获得 `.com` 地址。
    3. 本地 DNS → `.com` TLD → 获得 `server.com` 权威 DNS 地址。
    4. 本地 DNS → 权威 DNS → 获得目标 IP 地址。
    5. 本地 DNS 将 IP 地址返回给客户端。
