对于这类对象，不再使用 20 字节的基本对象名称，而是**使用另一个可变长度的整数**。同样，最高有效位 (MSB) 指示该整数的最后一个字节的位置。这个整数表示一个负偏移量——它告诉我们为了找到基本对象，需要在当前包文件中向后扫描多少个字节。负偏移量之后是经过 zlib 压缩的增量数据，就像 `OBJ_REF_DELTA` 一样。

### OBJ_OFS_DELTA（偏移量增量对象）的定义

OBJ_OFS_DELTA 是 Git 在 packfile 中使用的一种 **增量对象格式**，用于进一步节省空间。  
与 OBJ_REF_DELTA 不同，它 **不再使用 20 字节 SHA-1/SHA-256 引用基础对象**，而是使用一个 **可变长度负偏移量** 来指示基础对象的位置。
### 向前扫描
**从当前位置往文件头方向扫描，以锁定基础对象的位置。**在文件解析时，程序指针向着文件起始方向移动（即 offset 递减）
```
[type+size] [varint offset] [zlib delta instructions]
```
- 基础对象通过 **可变长度偏移量（varint）** 引用
- 偏移量通常只需 1～3 字节
- `[varint offset]` MSB格式
### 优势
- **节省大量空间**（1–3 字节 vs 固定 20 字节）
- **更快的 delta 解码**（无需哈希查找，偏移量可直接定位）
- **更稳定、更浅的 delta 链结构**
- **更好的压缩率**（更低熵，更易压缩）
	- delta object 之间的头部**模式高度重复**
	- 高位用于 continuation bit → 低位常有相同模式
	- 结果：packfile 的可压缩性上升 → 压缩率显著提高
- **更契合 packfile 排序优化（delta locality）**
	- Git 在打包时会对对象进行优化排序
	- 被频繁引用的对象（如最近修改的 blobs）
		- 偏移越小 → varint 越短 → 编码字节更少
