## 1. 核心观点  

`deltification` 是 Git 存储优化的核心机制之一，用于在对象数据库（object database）中通过**差异压缩（delta compression）**减少冗余数据。

## 3. 展开说明  
在 Git packfile 中，**负偏移量（negative offset）** 是一种指示相对位置的数值，用于 **OBJ_OFS_DELTA** 对象的基对象定位。

- 它不是绝对地址，而是 **相对于当前对象起始位置向后回溯的字节数**。
    
- 形式上是正整数，但语义上表示“向前/向上查找”，因此称作“负偏移量”。
    

用公式表示：

$$base\_pos=current\_pos−offset$$

其中：

- `current_pos` = 当前 delta 对象在 packfile 中的起始字节位置
- `offset` = variable-length 编码的整数（偏移量）
- `base_pos` = 基对象的位置
### 2. 为什么要用负偏移量

1. **节省空间**
    - OBJ_REF_DELTA 用 20 字节哈希标识基对象；
    - 对于包内对象，实际只需告诉 Git “向前多少字节就是基对象”，可用 1–5 字节的可变长度整数代替 20 字节哈希。
2. **顺序解析优势**
    - packfile 对象是 **按偏移顺序存储** 的；
    - 基对象总是位于 delta 对象之前，所以只需向后回溯即可找到。
3. **避免跨包依赖**
    - OBJ_OFS_DELTA 仅依赖当前 packfile 内的位置，**不需要全局哈希查找，提高解析效率。**
