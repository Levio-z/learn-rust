## 1. 核心观点  

由 Git 1.6 以后引入，通过服务端 Git 进程（`git-upload-pack` / `git-receive-pack`）与客户端协商所需对象，实现高效增量传输。

## 3. 展开说明  

- **定义：**  

    
- **原理流程（以 Fetch 为例）：**
    1. 客户端向服务器发送引用列表请求；
    2. 服务器返回分支与提交的引用信息；
    3. 客户端根据已有对象计算缺失对象；
    4. 客户端请求服务器打包缺失对象；
    5. 服务器生成 packfile 并返回；
    6. 客户端解包并更新仓库。
        
- **优势：**
    - 仅传输必要数据；
    - 可用于 Push / Fetch；
    - 支持差异压缩与多协议层封装。
