### `hlt` 指令的作用与内存安全性

在 x86 架构中，`hlt`（halt）指令是一条 CPU 指令，用于让处理器进入**低功耗等待状态**，直到下列事件发生之一：

1. 收到外部中断（如定时器、I/O 设备信号）
2. 系统复位或 NMIs（非屏蔽中断）
3. 特权指令触发异常（如双重故障）

与普通循环轮询不同，`hlt` 让 CPU **停止执行指令**，并且不会访问内存或修改寄存器状态，因此在使用 Rust 或类似安全语言封装时，可以做到**内存安全**：

- **没有破坏内存的风险**：`hlt` 不会读写任意内存，只是控制 CPU 状态。
    
- **低功耗休眠**：CPU 在 `hlt` 状态下几乎不执行任何指令，节约功耗。
    
- **无性能损失**：休眠期间 CPU 不浪费周期在空循环上，等待中断即可立即恢复，效率高。
