## 3. 展开说明  
### exit 命令 —— 使当前 Shell 退出

### 定义与作用

`exit` 是 Shell 的内建命令，用于让当前 Shell **退出当前执行环境**。退出时可指定一个无符号十进制整数 `n` 作为退出状态。

- 若当前处于 **子 Shell（subshell）** 中，调用 `exit n` 只会退出该子 Shell，返回到父 Shell 继续执行。
    
- 若当前是顶层 Shell，调用 `exit n` 会终止整个 Shell 会话。
    
- 当提供的 `n` 不在 **0–255** 之间时，退出状态是未定义的。
    

在 Shell 退出前，会执行已注册的 `EXIT` trap（除非 trap 内再次调用 `exit`）。

### 原理与规范细节

#### **退出状态规则**

- 指定 `n` 时：退出状态即为 `n`。
    
- 未指定 `n`：退出状态是“前一个命令的退出值”，若没有任何命令执行过，则为 **0**。
    
- 在 trap 中调用 `exit`：前一个命令是触发 trap 前执行的命令。
    

#### **特殊退出码解释（POSIX 保留）**

|状态码|含义|
|---|---|
|**126**|找到文件但无法执行|
|**127**|要执行的命令未找到|
|**>128**|命令被信号中断（128 + 信号编号）|

#### **错误处理（未定义行为的原因）**

POSIX 不规定 `exit` 参数错误或超过 255 的统一行为，这使得不同 Shell 可能：

- 截断到 0–255（如 bash、dash 部分版本）
    
- 直接报错并 exit 非 0
    
- 甚至 silently ignore
    

因此，**脚本中强烈建议只使用 0–255 的退出值**。

### 使用场景

#### 1. 正常退出程序

```sh
exit 0
```

#### 2. 表示执行失败

```sh
exit 1
```

#### 3. 在子 Shell 中进行错误传播

```sh
(
    command1 || exit 1
    command2 || exit 1
    exec command3
) > outputfile || exit 1

echo "outputfile created successfully"
```

#### 4. 在 trap 中处理退出逻辑

```sh
trap 'echo "cleanup"; rm -f tmpfile' EXIT
```

### ### 输出与输入规范

- **STDIN、输入文件、环境变量**：均不使用
    
- **STDOUT**：不输出任何东西
    
- **STDERR**：仅在发生诊断错误时输出
    
- **退出前执行 EXIT trap**
    

### ### 扩展知识点

- Shell 的退出值只能保存 0–255，因为它依赖 Unix 早期 wait() 系统调用的 8-bit 退出码字段。
    
- POSIX 不要求 Shell 返回完整的 waitid() exit code，因此高位内容丢失。
    
- 高于 128 的退出码通常用于信号编码，如：
    
    - Ctrl-C → 信号 2 → exit 130
        
    - Segfault → 信号 11 → exit 139
        

### ### 总结

`exit` 是 Shell 内建命令，用于终止当前 Shell 环境，并可携带 0–255 的退出状态。顶层 Shell 会完全退出，而子 Shell 只会结束当前子进程。退出前会触发 EXIT trap。由于不同 shell 在错误参数处理上历史差异较大，POSIX 特意将错误行为标记为“未指定”，使实现可自由处理。

### ### 学习方法论

1. **从行为到机制**：先理解退出码在 Shell/UNIX 进程体系中的意义，再学习命令行为。
    
2. **结合实际脚本阅读**：阅读开源 Shell 脚本理解退出策略如何设计。
    
3. **对比多 Shell 行为**：bash、dash、zsh 对 exit 错误参数处理不同，有助深入理解 POSIX 的未定义部分。
    
4. **结合 trap 机制学习**：exit 与 trap 的组合使用是脚本鲁棒性的关键。
    

### ### 练习题

1. 为什么 exit 只能返回 0–255？超过 255 会发生什么？

    ```sh
    foo() { false; }
    trap 'echo trap' EXIT
    foo || exit 3
    ```
    
3. 写一个脚本：在子 Shell 中执行三条命令，任何命令失败就退出并将错误码传给父 Shell。
    
4. 分析信号退出的编码方式：为什么 `SIGKILL` 会导致退出码 137？
