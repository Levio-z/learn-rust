## 1. 核心观点  
### Ⅰ. 概念层
- 通过一系列项目，你将构建一个网络键[值数据库](https://en.wikipedia.org/wiki/Key-value_database) ，支持多线程和异步 I/O。项目间隙，你将学习和练习完成下一个项目所需的各个科目。在这个过程中，你将探索多种设计及其权衡。


### Ⅱ. 应用层
### SOP
- 构建前需要学习[building blocks](#building%20blocks)
- 构建新项目：从上一个项目复制源代码是可以的
- cargo test是项目完成标志
- 项目结束打tag
	- TP-201-P1-xx
>你**不应该在完成项目之前去读任何示例代码** ——良好的学习需要自己尝试并失败，直到成功为止。不过，我们鼓励你从中学习，并事后将这些技巧应用到自己的项目中。但请记住，示例项目并不是解决问题的唯一方式——甚至不是最好的方式。相信自己和自己的创造力。

## 3. 展开说明  
- 每个项目都是前一个项目的延伸，每个项目都是在上一个基础上构建的——API、命令行接口和实现部分内容在不同项目之间通常保持不变，而你只会根据项目主题集成新功能和改进。因此，每个项目开始时，从上一个项目复制源代码是可以的。
### building blocks
- 包括外部阅读材料、外部编程问题以及其他单一学科内容。
- 是一个清理思绪、暂时放下大项目、专注于单一学科学习和练习的机会。 **不要错过。**
